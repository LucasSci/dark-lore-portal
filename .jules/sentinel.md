## 2024-03-27 - Weak random number generation for security purposes
**Vulnerability:** Weak, predictable random number generation was found in `src/lib/virtual-tabletop.ts`, `src/lib/sheets/runtime.ts`, `src/lib/vtt-realtime.ts`, and `src/lib/vtt-assets.ts`. Developers were using `Math.random().toString(36)` (often concatenated with `Date.now()`) to generate identifiers for tabletop entities, scene events, sheet runtime connections, and asset items.
**Learning:** These IDs are essential elements that connect players and resources during a VTT session. Predictable ID generation via pseudo-random functions like `Math.random()` opens up a security vulnerability, potentially allowing bad actors to guess connection, entity, or presence keys, leading to unauthorized state modifications or data leakage.
**Prevention:** For any random identifier generation, particularly when related to application entities or connections, avoid `Math.random()`. Instead, consistently use cryptographically secure random number generation wrappers like `generateSecureId` and `generateSecureShortId` added in `src/lib/utils.ts`.
